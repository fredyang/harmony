Get started

Harmony is a lightweight jQuery based library for building trivial widget or complex html application. It implements publish-subscribe pattern, and provides an environment where model and view can live in harmony. Here are some of its features.

  1. Lightweight client-side model repository
  2. Unified subscription for both view and model
  3. Subscription can be registered declaratively, programmatically, or mixed.
  4. Reusable, functional, asynchronous event handler implemented by workflow and activity.
  5. Bundled with lots of reusable, composable plugins.

Harmony is hosted on Github, it is available for use under the MIT software license.

How Harmony works?

Unlike other MVC framework, where controllers are in control, in Harmony, view and model take control of themselves. Below are some simple rules the library follows.

  1. View and model are all objects.
  2. Every object controls itself.
  3. No object control any other object.
  4. An object can trigger events of himself.
  5. Objects can react to other objects' events.
  6. Objects do no act.

The reason to follow to these rules is that in a stateful client side environment, a controller tends to control too many things which are in conflict with each others. The effect to synchronize the state and behavior of others tends to be huge and sometimes futile. When designing client application, we normally think like, "When this happens, I want to change this like this". Harmony use another mindset, "When this happens, I want to change myself like this". It is much easy to change myself than to change others.

Can I use controller?

Yes, as long as it controls well. For whatever reason, such as performance, directly manipulating others is still required in some cases, you, developer control what needs to be done.

Can I use with Harmony other MVC frameworks/libraries?

Yes. In Harmony, objects without subscriptions will do nothing, objects with subscriptions will only change themselves not others, so they are very harmless. Harmony is just a library about subscriptions, and it plays well with others.
